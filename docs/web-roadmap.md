# Web Roadmap

## 第一階段

- 完成首頁與公司詳細頁的正式版視覺語言。
- 保持首頁的資料表概況、公司查詢與真資料說明。
- 保持公司詳細頁的公司基本資料與日月年報最新摘要。
- 整理共用 layout、按鈕、卡片、表格與 empty state 樣式。
- 補齊 Web 啟動、venv 與依賴說明。
- 確保首頁、公司頁與 API 都能在現有 SQLite 結構下穩定運作。

## 第二階段

- 依現有資料表延伸市場排行榜頁，例如成交量排行或資料更新狀態頁。
- 強化個股頁的報表切換與更多欄位呈現。
- 若後端有可靠資料來源，再補新聞、評論或分析欄位。
- 如果未來要做 watchlist，先補使用者模型與偏好資料表，再做 UI。
- 視需求評估是否引入更完整的前端建置工具，而不是先為設計稿導入過重堆疊。

## 不在目前範圍

- 假新聞卡片
- 假分析師評級
- 假交易按鈕流程
- 沒有資料來源支撐的市場指標與財務比率
