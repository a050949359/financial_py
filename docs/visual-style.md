# Web Visual Style

## 定位

- 網站定位為資料導向的 TWSE 閱讀介面，不是交易平台，也不是財經媒體首頁。
- 視覺方向採 editorial market console：有編輯感、有資訊節奏，但內容仍以真資料為主。

## 設計原則

- 先展示資料可用性，再展示內容。
- 沒有資料的區塊要直接標示 unavailable 或尚未匯入，不補假圖表、假新聞、假指標。
- 首頁、個股頁與未來擴充頁面要共享同一套品牌語言，不各長各的。

## 視覺語言

- 背景：冷色系亮底，搭配低對比網格與模糊光暈，營造資料桌面感。
- 版面：使用大標題、分節卡片與留白，保留閱讀節奏。
- 卡片：以半透明或淺色 surface 呈現，圓角偏大，陰影克制。
- 表格：保留表格結構，但外層用卡片包起來，避免純資料庫介面感。

## 色彩

- 主色：藍色系，用於品牌、主按鈕、主要導覽。
- 強調色：較深藍色，用於文字連結與次級互動。
- 成功色：綠色，用於資料可用或正向狀態。
- 中性色：灰藍色，用於說明文字、邊界與背景層次。

## 字體

- 標題使用較具識別性的 serif 字體，營造編輯感。
- 內文與資料使用無襯線字體，優先考量中文可讀性。
- 數值與標籤沿用內文字體，但以字重與大小區分層級。

## 元件規則

- Hero：一頁只保留一個主 hero，負責交代資料來源與頁面價值。
- Summary cards：只放真實摘要，不放人工估值或推論。
- Search：首頁保留一個主搜尋區，採 Jinja 首屏加 Vue 局部更新。
- Detail cards：公司頁優先展示公司基本資料與最新報表摘要。
- Empty state：清楚告知需要先執行哪個 make 指令補資料。

## 內容規則

- 允許：公司基本資料、最新日月年報、資料表統計、匯入時間。
- 暫不顯示：本益比、殖利率、目標價、評級、新聞、技術圖形，只因目前資料流無法支撐。
- 如果未來新增內容，需先確認有可靠來源與落地資料表，才能進頁面。

## 前端技術規則

- 目前以 Flask + Jinja 為主。
- Vue 只用在局部資料綁定，不做整站 SPA。
- 樣式先收斂在 web/static/site.css；除非 UI 規模顯著擴大，否則不急著引入完整 Tailwind toolchain。
